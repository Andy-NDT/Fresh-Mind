# Changelog

Все заметные изменения в проекте Fresh Mind документируются здесь.
Формат: [Keep a Changelog](https://keepachangelog.com/ru/1.1.0/).

## [0.3.3] — 2026-06-06

### Исправлено

- **Секторы групп «уезжали» при удалении сферы.** Раньше углы всех сфер делились
  поровну на полные 360°, а белый крест и границы четырёх групп (Здоровье /
  Общество / Труд / Развитие) были прибиты к фиксированным 90°-меткам. Совпадало
  только когда в каждой группе было ровно по 5 сфер; стоило удалить одну — все
  клинья сползали с креста.

### Изменено

- **Адаптивные секторы внутри групп.** Теперь каждая группа держит фиксированный
  квадрант 90° (крест вертикально-горизонтально не двигается), а сферы внутри
  группы делят эти 90° поровну — адаптивно к их количеству. Удаление сферы
  перебивает только её собственный квадрант, остальные три стоят на месте. Даже
  одна оставшаяся сфера растягивается на весь квадрант. В сбалансированном случае
  (по 5 сфер) колесо выглядит ровно как раньше — без визуальных изменений.

## [0.3.2] — 2026-06-03

### Изменено

- **Адаптивный дашборд на широком окне (от 1000px).** Под колесом теперь два
  столбца: слева — карточки сводки (Здоровье / Общество / Развитие / Труд +
  метрики), справа — «Динамика» и «Сводка». На узком окне всё схлопывается в
  один столбец, как раньше (Динамика и Сводка уезжают вниз). «Записи» — во всю
  ширину под сеткой.
- **Колесо — по центру и крупнее контент-блоков.** Размер колеса ограничен и по
  ширине, и по высоте окна; карточки сводки, Динамика и Сводка сужены, чтобы на
  фуллскрине не растягивались и не «забивали» колесо.

### Исправлено

- **На фуллскрине колесо занимало почти весь экран**, и блоки под ним (карточки
  сводки, строка метрик) уходили за нижний край области прокрутки — их не было
  видно. Высота колеса теперь привязана к высоте окна, нижние элементы видны.

## [0.3.1] — 2026-06-03

### Добавлено

- **Кнопка «× убрать оценку» на колесе** — снять оценку любой сферы:
  открой сферу → кнопка появляется под колесом. Снятие можно отменить (↶).

### Изменено

- Числа под колесом (сводка, дельты, динамика, счётчик «N / M сфер»)
  обновляются мгновенно при любой оценке — и из быстрого ввода, и кликом
  по самому колесу.

### Исправлено

- **Оценка сферы из редактора заметки не выставлялась.** Клик по цифре в
  шкале сферы закрывал шкалу, и ничего не происходило — обработчик «клик
  снаружи» перехватывал нажатие до того, как срабатывал выбор. Теперь выбор
  регистрируется и сразу пишет сегодняшнюю оценку на колесо.
- **Время уведомления не выбиралось в настройках.**
- Дата оценки и маркеров записей считается по локальной дате, а не UTC —
  вечерняя оценка у часовых поясов с плюсовым сдвигом больше не падает на
  предыдущий день.

## [0.3.0] — 2026-06-01

### Добавлено

- **Шеринг PNG** — кнопка-скачать в трёх местах: колесо за день,
  сравнение двух дат, график динамики. Открывает модал с выбором размера
  (1080×1080 квадрат / 1920×1080 широкий / 1200×630 горизонтальная карточка),
  фона (прозрачный / белый / тёмно-фиолетовый) и опциональной подписи.
  В углу картинки — фирменный маркер **Fresh Mind**. Через нативный системный
  диалог сохранения файлов Electron.
- **Динамика — недельный режим для длинных периодов** (≥180 дней). Дневные
  точки сливались в кашу — теперь агрегация по неделям с богатым тултипом:
  средние по 4 группам сфер + дельты к предыдущей неделе, индикатор малой
  выборки (<3 дней с оценками → пунктир + opacity).
- **Адаптивный размер точек** по плотности графика (sparse / medium / dense)
  через CSS-переменные. Точка по-прежнему читается и при 365 днях.
- **Пассивные weekly-маркеры** над неделями с топ-настроением (эмодзи) и
  счётом записей.
- **Дельты в плитках «Выше всех / Ниже всех»** на дашборде — против вчера,
  fallback на последнюю оценку сферы. Зелёным/красным/серым ('—' если нет
  истории).
- **Фильтр по диапазону дат в ленте записей** — кнопка-календарь рядом с
  pin-фильтром, popover с двумя `<input type=date>`.

### Изменено

- **Иерархия заголовков** «Динамика / Сводка / Записи» унифицирована — все
  `color: var(--fm-deep)`.
- «Общая средняя» → «Динамика», «Сводка за май 2026» → «Сводка май, 2026».
- Шеврон рядом с заголовком сводки убран, hover-индикатор через `::after`
  с rotate-анимацией.
- Хинт «ЛКМ — оценка · ПКМ — детали» переехал под колесо.
- Кнопки «сегодня» + «сравнить» вынесены во второй ряд под датой
  (`.radar-datebar-secondary`).
- Кнопка отмены ↶ всегда видна (disabled когда стек пуст).
- **EntryFeed: сохранение абзацев** — `previewFromText` больше не сжимает
  `\n` в пробелы.
- **EntryFeed: edit-режим** — кнопки [Отмена] [Сохранить] сверху справа над
  textarea. Enter — сохранить, Shift+Enter — новая строка, Esc — отмена.
  Read-only sphere chips скрыты в edit-режиме.
- «Отмечено сегодня» → «Отмечено сфер» (корректно для любой даты).
- **HTML-маркеры записей** на графике вместо SVG `<text>` (не плющатся под
  `preserveAspectRatio="none"`).
- Tabs графика динамики (`.trend-tab`) теперь адаптивно сжимаются — на
  узких окнах не ломают разметку при добавлении share-кнопки.

### Исправлено

- **React Hydration Error**: `.sp-header` был `<button>` с вложенным
  `<button class="sp-mode-btn">` — заменён на `<div role="button">` с
  keyboard handlers.
- Удалён мёртвый `OverviewPanel.{jsx,css}` — был дубликатом `DashboardSummary`.
- На Win11 включены закруглённые углы главного окна (`roundedCorners: true`).

### Технические заметки

- Новые модули: `src/renderer/shared/sharePng.js` (хелпер html2canvas),
  `src/renderer/shared/ShareDialog.{jsx,css}` (переиспользуемый модал).
- Новый IPC: `save-png-file` — main принимает `Buffer` + suggestedName,
  открывает нативный `dialog.showSaveDialog`, пишет PNG через `fs`.
- Новый preload-метод `savePngFile(bufferOrUint8, suggestedName)` —
  нормализует `ArrayBuffer` / `Uint8Array` перед IPC.
- 2 новых db-helpers: `getWeeklyAverages` (с `group_id`), `getWeeklyEntryStats`
  (с `topEntries` / `moods`).
- 1 новый db-helper для дашборд-дельт: `getLastRatingsBefore` + правка
  `getRatingsForDate` под fallback-цепочку.
- `html2canvas@^1.4.1` добавлен в `dependencies` (установлен с
  `--legacy-peer-deps` из-за конфликта `@emoji-mart/react` ↔ React 19,
  существующего в проекте).
- В UI и коде нет упоминаний конкретных соц-сетей — размеры экспорта
  нейтральные (квадрат / широкий / горизонтальная карточка).

## [0.2.0] — 2026-05-29

### Добавлено

- **Окно «Экспорт для ИИ-аналитики»** — отдельная кнопка в Настройки → Бэкап и
  данные. Открывает попап с выбором периода, типа разбора и параметров.
- **6 готовых типов разбора:** глубокая ретроспектива, анализ одной сферы,
  помощь с решением, поиск слепых пятен, сравнение двух периодов, свой промпт.
  Каждый — нейтральный шаблон, не упоминающий конкретные ИИ-сервисы.
- **Структурированный отчёт `.txt`** с 6 секциями:
  1. Шапка с totals (записи, оценки, активные дни)
  2. Динамика сфер по группам с трендом первой → последней трети периода
  3. Топ-10 ярких записей (взвешивание: pinned ×3 + наличие настроения +
     кол-во сфер ×0.5 + длина текста)
  4. Паттерны: время записей по 4 бакетам, топ-теги, распределение настроений
  5. Корреляции Pearson между сферами (показываются только при ≥60 днях с
     оценками — гейтинг от ложных сигналов на малых выборках)
  6. Полные записи в хронологическом порядке
- **Зашивка промпта в начало файла** — когда выбран тип разбора, инструкция
  пишется в шапку файла перед данными. Один upload в ИИ-сервис разбирает весь
  цикл без необходимости копировать промпт отдельно.
- **Стримовая генерация** — отчёт пишется в файл секциями через
  `fs.createWriteStream`, без аккумулирования в памяти. Размер за год данных
  не нагружает RAM.
- **Живой счётчик записей** под выбором периода — обновляется при смене
  периода через IPC `count-entries-in-range`.
- **Период «Всё время»** использует дату первой записи, а не сентинел `2000-01-01`.

### Изменено

- В README раздел «Что отличает Mind» расширен описанием AI-экспорта.
- Roadmap-секция в README: v0.2.0 помечен как выполненный.

### Технические заметки

- 5 новых db-helpers: `getSpherePeriodStats`, `getRatingsByDateAcrossSpheres`,
  `getTimeBuckets`, `getDaysWithRatingsCount`, `listEntriesInRange`,
  `getFirstEntryDate`, `countEntriesInRange`.
- Новый модуль `src/main/aiExport.js` — генератор отчёта с streaming-API.
- Новые IPC: `open-ai-export`, `close-ai-export`, `resize-ai-export`,
  `count-entries-in-range`, `get-first-entry-date`, `export-ai-report`.
- Аудит чистоты: ноль упоминаний LLM-сервисов (Claude/ChatGPT/Anthropic/OpenAI
  и др.) в коде UI и шаблонах промптов.

## [0.1.0] — 2026-05-29

Первый релиз Fresh Mind.

### Добавлено

- Колесо 20 сфер жизни с шкалой оценок 0–10
- Дневник записей с TipTap-редактором: форматирование, теги через `#`,
  эмодзи-настроение, прикрепление картинок
- Группировка сфер (Здоровье / Развитие / Труд / Общество) с цветовыми
  секторами фона
- Перемотка дат через ползунок + сравнение двух дат (наложение паутинок)
- Полнотекстовый поиск через SQLite FTS5
- «В этот день» — записи из этой же даты в прошлые годы
- Pinned-памятки сверху ленты
- Корзина с восстановлением soft-deleted записей
- Экспорт / импорт JSON, полный бэкап папки данных
- Трей-интеграция, автозапуск с Windows, попап-напоминание (по умолчанию 22:00)
- Onboarding для первого запуска
- Окно «О приложении» с прогрессом ведения дневника
- Единая анимация появления окон (fmWindowAppear)
- Error boundary вокруг каждого renderer entry point с возможностью
  «Перезапустить» через IPC
- Future-schema protection — приложение прерывает запуск с понятным dialog'ом,
  если БД создана более новой версией приложения (защита от downgrade)

### Стек

- Electron 35 + electron-vite + React 19 (JavaScript, не TypeScript)
- better-sqlite3 для локального хранилища
- node-window-manager для стэкинга попапов с Fresh Ear/Eye
- electron-builder с NSIS + portable target
